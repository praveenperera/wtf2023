# Agent Instructions

- Do not use hand-drawn SVGs when Recharts or lucide can cover the need
- Always prefer ready-made components
- Always use existing shadcn components when available
- Public-facing copy must read as final reader-facing site content, not notes about how the page was built, migrated, restored, sourced, or structured.
  - Do not mention preservation, legacy status, CMS/source migration, original pages, local assets, first-party assets, redirects, copied screenshots, memes, receipts, restoration rationale, fallback states, committed/local/generated data, source-backed/source sections, page routes, UI wrappers/cards/panels, or implementation details unless the user explicitly asks for that wording.
  - Prefer article-native labels and claims, e.g. "Article images", "Evidence", "Read full rebuttal", "Methodology", or omit labels entirely.
  - Before finishing copy work, search rendered/source copy for meta words like `page`, `route`, `fallback`, `local`, `committed`, `generated`, `screenshot`, `receipt`, `source-backed`, `wrapper`, `panel`, `legacy`, and `migration`, and rewrite any reader-visible hits unless they are genuinely domain terms.
